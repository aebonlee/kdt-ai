## 1) 콘텐츠 — A+
- **규모**: 18과목/42일차, 실습예제 **317개·코드 5,466줄**(1차 대비 실습 코드량 약 71% 증가), 강의 데이터 13,000여 줄. 평가기준 12과목(실습목적·구성·배점·제출물), 복습 퀴즈 20문항(해설 포함).
- **근거성(핵심)**: 로컬 자료 폴더의 4기 교육생 교재·종합실습 가이드(Practice·Checkpoint)·3기 교재·ml-dl 전용 덱·평가양식까지 **전수 추출·반영**. 실라버스와 대조 확증. 원문 verbatim 없이 우리말 재서술로 저작권 안전.
- **자립성**: 교시별 시간표+이론+개념+상세+실습+예제+과제+평가기준+퀴즈 — "이 사이트만으로 8시간 수업" 준거를 실습·평가 축까지 확장 충족.
- 잔여 과제: 퀴즈가 2과목(Python·VectorDB)에만 존재 — 나머지 16과목 확장 여지.

## 2) 기능 — A+
- 학습자 여정 전주기: 선수학습(19주제, SKALA 연계 안내) → 일정(캘린더·지역필터) → 학습강의안(9섹션) → **실습교안**(과목별 페이지·인쇄·코드 복사·녹색 주석) → 평가기준·퀴즈 → 학습관리(자가 진도) → 게시판·대시보드(OAuth).
- 산출물 3종 동시 유지: 사이트 · 정적 실습교안(빌드마다 자동 최신화) · A4 PDF 교재(400p+) — 단일 데이터 소스에서 생성.
- 담당 외 SKALA 과목은 선수과정/후속·심화 멘트로 참고자료·선수학습에 통합 — 과정 전체 맥락 제공.

## 3) UX·모바일 — A
- **측정 기반 검증**: 전 라우트 scrollWidth 계측 → 오버플로 3곳(일정 836·강의안 744·팀 687) 확정 수정 → 전 페이지 `scrollW==vw`. 전역 `overflow-x:clip` 안전망.
- 모바일 패턴 통일: 사이트 헤더(≤880px)·실습교안(≤820px) 모두 햄버거 드로어. 실습교안 코드는 기본 줄바꿈+가로 토글로 제스처 부담 제거.
- 라이트/다크 전 화면 대응(로그인 카드 포함), 브랜드 일관성(인디고 칩 로고).
- 감점: 실기기(<500px) 검증은 헤드리스 한계로 사용자 확인 의존, 실습교안 단일 HTML 1.5MB는 모바일 첫 로드에 부담(분할 여지).

## 4) 성능 — A
- 초기 로드 gzip: index 19.5KB + react-vendor 53.5KB + CSS 5.9KB ≈ **79KB**. Supabase(54.4KB)·과목 데이터는 지연 청크. 최대 과목 청크 34.9KB(ml-dl).
- 빌드 1.6s(분할 스크립트 포함), 강의안 첫 진입은 성능 분할 체계(21편) 유지.
- 여지: 실습교안 HTML 過大(전 과목 인라인) — 과목별 분할 로드 또는 gzip 서빙 확인.

## 5) 코드 품질 — A
- 데이터(진실원본 lecture*.js·exams·quizzes) ↔ 렌더(React/생성기) 분리, `examplesFor` 3층 병합, 생성물 전부 gitignore+prebuild 자동화 — 콘텐츠 수정→빌드만으로 3개 산출물 동기화.
- 이스케이프 규율(코드 내 리터럴 `<>&`, 백틱·${} 금지)이 파이프라인에 명문화되어 이중 이스케이프 0건.
- 감점: 자동화 테스트(단위/E2E) 부재 — 회귀는 수동 스크린샷·측정에 의존. ESLint 등 린트 체계 미도입.

## 6) 운영·배포 — A+
- main 푸시 → Actions 자동배포, 최근 전 배포 success(약 30초). prebuild가 split+실습교안 생성까지 수행하므로 CI에서 산출물 누락 불가.
- 문서화 26편(Dev_md)으로 이력·구조·의사결정 추적 가능. 메모리에 자료 자동반영 상시 흐름 기록.
