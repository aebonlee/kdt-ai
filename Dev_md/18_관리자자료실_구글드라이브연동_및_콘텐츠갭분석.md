# 18. 관리자 자료실(구글드라이브 연동) + 강의안 콘텐츠 갭 분석

## 배경
로컬 `docs/`(SKALA 4기 실습교수 강의자료 97개, 561MB — PDF·PPTX·XLSX·ZIP)를
관리자(강사)만 사이트에서 바로 열람하고 싶다는 요구.

### 왜 리포에 직접 넣지 않았나 (설계 결정)
- **공개 노출**: skala 리포는 public + GitHub Pages(skala.dreamitbiz.com). `public/`에 넣으면
  관리자 로그인과 무관하게 URL만 알면 누구나 다운로드 → SK 소유 교재·타 강사 실명 평가양식 유출.
- **용량**: 561MB 바이너리를 public git에 커밋하면 히스토리 영구 오염, GitHub 100MB 파일 차단.
- **렌더링**: PPTX·XLSX(45개)는 브라우저 내장 뷰어로 못 염.

→ **결론: 파일은 비공개 구글드라이브에 두고, 리포엔 "링크(파일ID)"만 저장.**
   구글이 접근제어(구글 로그인) + 관리자 UI 게이트 = 이중 보안. PDF·PPT·Excel 모두 구글 뷰어로 미리보기.

## 구현
- **원본 드라이브 폴더**(비공개, owner: aebon@hs.ac.kr)
  `https://drive.google.com/drive/folders/1bzElDI2zY-2qGQpE6vamqEg7KLRO4qn5`
- `src/data/docs.js` — 97개 파일 카탈로그(5개 그룹, 파일ID·유형·용량). preview/view/download URL 헬퍼.
- `src/components/RequireAdmin.jsx` — `ADMIN_EMAILS`(aebon@kakao.com / aebon@kyonggi.ac.kr) 게이트.
- `src/pages/Admin.jsx` — 검색 + 카테고리 칩 + 2단(목록/드라이브 preview iframe) 레이아웃, 새 탭·다운로드.
- `App.jsx` — `/admin` 라우트(`RequireAdmin`), lazy 청크.
- `Header.jsx` — 관리자에게만 `🔒 자료실` 네비 노출.

### 파일 열람 URL 규칙
- 미리보기: `drive.google.com/file/d/{id}/preview`
- 새 탭: `.../view` · 다운로드: `drive.google.com/uc?export=download&id={id}`

### 운영 메모
- 미리보기가 안 뜨면 브라우저에 강사 구글계정(aebon@hs.ac.kr) 로그인 여부 확인.
- 드라이브에 파일을 추가/교체하면 `docs.js`에 파일ID를 추가해야 목록에 반영됨.

## 강의안 콘텐츠 갭 분석 (공식 4기 실라버스 ↔ 사이트 curriculum.js)
사이트는 이애본 강사 담당 20과목을 다룸. 공식 실라버스 PPTX와 대조해 **추가 권장 토픽** 도출.
※ 본 분석은 커리큘럼 개요(curriculum.js) 기준. 상세 강의안(lecture*.js)에 일부 이미 포함됐을 수 있음.

### 우선순위 A — 구조적으로 빠진 핵심(현재 사이트에 거의 없음)
1. **MCP (Model Context Protocol)** — 공식 Capstone의 중심. MCP Server 구성(Tool·Resource·Prompt 분리),
   MCP Inspector, FastAPI+RAG+Agent+MCP Client 통합. 사이트 Capstone은 일반 서술뿐 → 대폭 보강 필요.
2. **Streaming(SSE) · Multi-Agent Streaming · Vercel AI SDK** — Capstone. 사이트 없음.
3. **Agentic RAG** — VectorDB·RAG·Agent 실라버스에 반복 등장. 사이트는 클래식 RAG만.
4. **Production/Ops 관점** — Observability & Eval, Cost & Latency, Error Handling, Stateless Session,
   Query Routing / Validator Agent / Conditional Routing / Dynamic Planning (Capstone). 사이트 없음.

### 우선순위 B — 기존 과목 심화
5. **LLM 심화(transformer Day2)**: Scaling Law, RLHF, MoE, In-Context Learning,
   Open-weight vs Closed model, 벤치마크·Leaderboard, Software 1.0/2.0/3.0 패러다임.
6. **RAG 고급 리트리버**: ParentDocumentRetriever, MultiQueryRetriever, Ensemble, LongContextReorder,
   SemanticChunker, Naive→Advanced→Modular→Agentic 4단계, RAG Evaluation pipeline.
7. **VectorDB(수정본)**: Re-ranking, Hybrid Search 심화, Agentic RAG, Production Architecture,
   Qdrant(FAISS→Qdrant), 최신동향(MEMO: Memory as a Model).
8. **LangChain 심화**: Runnable 인터페이스 통일성(왜 LangGraph Node가 Runnable인지), bind_tools() 메커니즘,
   RunnableParallel/RunnableLambda/RunnableWithMessageHistory, LangChain vs LangGraph 경계.
9. **sLLM 심화**: MLM vs CLM 구조 비교, PEFT 변형 비교(LoRA·Adapter·Prefix/Prompt tuning),
   Use Case 시나리오(DBMS·영업비밀·개인정보), sLLM+VectorDB 서비스 파이프라인.
10. **Agent 심화**: Agent Protocol, Agentic Workflow(Goal·Plan·Execute·Reflect),
    Harness Engineering, Parallel Execution/Fan-out, Supervisor·Middleware 패턴.

### 우선순위 C — 공통 프레이밍
11. **Harness Engineering** — prompt·agent 실라버스 공통 개념. 사이트 미언급.
12. **Prompt Day 과정 정합** — "Prompt Engineering을 넘어서 → Context Engineering → Agent/Harness" 흐름.
