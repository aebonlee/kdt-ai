## 2. A안 구현 — 데이터 이동 0건, 필드 + 렌더 분기

### 데이터 (이동·삭제 없음)
`origin: 'practice'` + `source` 두 **선택 필드**만 추가. 없으면 주강사 계열로 동작하므로 **기존 카드는 손대지 않았다.**

`git diff` 검증: **60 insertions / 20 deletions**. 삭제 20줄은 각 카드의 마지막 속성 줄이 쉼표를 달고 재출력된 것뿐(내용 동일). 즉 **순수 필드 추가**다.
- 예제 총계 215 → **215** (불변), otherdeep 개념 115 · 실습 76 (불변)
- merge 스크립트 경유(시드 재생성 → 패치 스크립트 → 재병합). `adb9212`·`854b2a4` 이후로 시드를 다시 떠서 코디네이터 커밋을 덮지 않았다.

### 렌더 (`scripts/build-textbook.mjs`)
```js
const isPractice = (x) => x?.origin === 'practice'
const notPractice = (x) => !isPractice(x)
function renderPracticeSupplement(concepts, examples)   // 신규
```
- 담당 과목 `renderDay`: 본문 실습예제에서 practice 제외 → **마지막 일차**에 과목 전 일차의 practice 카드를 모아 소절 출력(퀴즈 다음)
- 담당외 `renderEtcCourseBlock`: 핵심개념·따라하기실습에서 practice 제외 → **블록 말미**에 개념+실습을 함께 소절 출력
- 소절 제목 **"🧩 실습교수 보충자료"**, 그 아래 `<p class="note thin">출처 — …</p>` 한 줄

### 출처 표기의 성격 (대표 정정 반영)
자료 대부분이 AI 생성물이라 **저작 크레딧이 아니라 대표가 인용 관계를 되짚기 위한 관리자용 메타정보**다. 따라서:
- 기존 제목 괄호 표기(`… (판교 7반 김일한 교수 자료)`)는 **그대로 두고 늘리지 않았다**
- 새 출처는 **카드 데이터 필드(`source`) + 소절 머리 한 줄**로만. `.thin` 스타일이라 화면에서 튀지 않음
- 구조화는 `origin` 붙는 20건에만 적용. 전 카드 소급 부여는 하지 않음

## 5. docs.js ↔ Dev_md 연결 진단 (요청: 진단만)

### 현황
| 축 | 위치 | 규모 | 성격 |
|---|---|---|---|
| **받은 원본** | `src/data/docs.js` | 12그룹 **164건** | 항목 필드 `t`(제목)·`id`(드라이브)·`x`(확장자)·`b`(크기)·`d`(날짜) |
| **반영 이력** | `Dev_md/` | 53편 | 산문. 사람만 읽을 수 있음 |

### 진단: **끊겨 있다. 총괄 집계의 가장 큰 빈 곳이 맞다.**
- `docs.js` → `Dev_md` 참조 **0건**. `docs.js` 항목에 "어디에 반영했는지" 필드가 **없다**
- 따라서 사이트만 보고는 **"이 파일이 반영됐나, 안 됐나"를 알 수 없다**. 164건 중 미반영분이 몇 건인지 집계 불가
- 역방향(카드 → 원자료)은 이번 `source` 필드로 **20건만 부분 확보**. 나머지는 제목 괄호 표기에 의존

### 최소 변경 제안 (만들지 않음 — 대표 판단 대기)
`docs.js` 항목에 **선택 필드 `ref` 하나** 추가하는 것이 최소 변경이다.
```js
{ t: '…', id: '…', x: 'pdf', b: 5996327, d: '2026-07-15',
  ref: ['otherdeep.stats', 'Dev_md/48'] }   // ← 추가 (없으면 미반영)
```
- 관리자 자료실 화면에서 `ref` 있으면 **반영됨 배지**, 없으면 **미반영**으로 자동 집계 → "무엇이 아직 반영 안 됐나"가 한 화면에 보임
- 스키마 호환(선택 필드) · 스모크 무관 · 화면 1곳만 수정
- 소급 입력은 164건 전부 할 필요 없이 **날짜 그룹 단위로 점진 부여** 가능
- 이 방향은 **과목 × 분반 × 날짜** 단위와도 맞는다(학생 개인 추적형 LMS 기능은 이 운영 구조에서 성립하지 않으므로 제안하지 않음)
