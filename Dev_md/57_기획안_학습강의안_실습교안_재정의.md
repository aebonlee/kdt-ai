# 57. 기획안 — 상단메뉴 「학습강의안 / 실습교안」 재정의

작성: 2026-07-24 (대표 지시: 기획안 먼저 보고, 승인 후 구현)

## 1. 문제 정의 — 지금 무엇이 안 맞나

현재 사이트는 두 메뉴가 **같은 콘텐츠를 정렬만 달리해** 보여준다(7/23 대표 지적 "왜 연관되게 가져가지"의 근본 원인).

| | 현재 | 문제 |
|---|---|---|
| 학습강의안(/lectures) | **날짜(세션) 중심** | 과목 전체를 이어서 학습하기 어렵고, 교재 그라운딩 심화가 일부 과목(transformer2)에만 있음 |
| 실습교안(/textbook) | 과목 중심이지만 **강의안과 동일 구성** | 대표의 실전 수업 기록(평가·맞춤 변경·반별 차이)이 담기지 않음 |

## 2. 재정의 — 두 문서의 역할 (대표 지시 반영)

### 학습강의안 = "배우는 문서" (과목 중심)
- **교과목별로 정리** — 전과목 마스터 A표 30과목 순서(진행순)
- 각 과목 = 일차별 학습목표 · 핵심개념 · **교재에 입각한 심화 이론(페이지 근거)** · 상세 학습내용 · 예제 · 복습퀴즈 · 추가학습
- 표준 모델 = 이번에 완성한 **transformer2 수준**(교재 107p 발췌 심화 8편·개념 8종·도해 SVG)
- 학생이 과목 하나를 처음부터 끝까지 다시 학습할 수 있는 완결성

### 실습교안 = "수업 실전 기록" (세션·시간 중심)
- **개강일(7/14)부터 시간순** — 이애본 배정 캘린더(마스터 I표, 20개 구간) 그대로
- 각 세션 카드 = 날짜·캠퍼스·반(호실)·전임교수 + **그 수업에서 실제로 한 것**:
  - 실습 진행 내용 (그 반 맞춤으로 변경한 사항 명시 — 예: 2반 마감 변경, 팀 편성, 템플릿 추가 배포)
  - **평가** — 평가기준·배점·제출 방식·마감 (반별 상이한 것 그대로: 7반 이해55/해석45 vs 2반 실습1 40/60+실습2)
  - 제출·채점 현황 (관리자만: 제출 수·피드백 이력)
- 같은 과목이라도 **반마다 다른 카드** (7반 Transformer ≠ 2반 Transformer — 이미 분리한 원칙의 확장)

한 줄 구분: **학습강의안은 "무엇을 배우나", 실습교안은 "그날 그 반에서 실제로 무엇을 했고 어떻게 평가했나".**

## 3. 정보 구조(IA) 설계

```
상단메뉴:  홈 | 학습강의안 | 실습교안 | 참고자료 | 부록 | ...

학습강의안 /lectures
 ├ 좌측: 과목 목록 (마스터 A표 진행순, 담당/타강사 구분 탭)
 ├ 본문: 과목 → 일차별 (현 Textbook의 과목 뷰를 계승·개선)
 └ 날짜 진입(/lectures/2026-07-23)은 유지하되 → 해당 과목·일차로 리다이렉트

실습교안 /practice (신설 라우트)
 ├ 좌측: 세션 타임라인 (7/14 개강부터, 완료/오늘/예정 상태 표시)
 ├ 본문: 세션 카드 = 수업 개요 + 실습 진행 + 맞춤 변경 로그 + 평가 + (관리자) 채점 현황
 └ 데이터: sessions 확장 — 세션별 practiceLog(변경사항)·evalRef(평가안)·guideRef(안내)
```

              '발표과제 템플릿(fashion_coordinator) 추가 배포' ],
