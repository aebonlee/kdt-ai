## 2. 영역별 평가

### 2-1. 콘텐츠 완성도 (19/20)
- 18과목 × 일차 = **42 day-key 전부**가 교시표·실습·핵심개념·심화이론·예제·실전소스·상세항목·Lab·과제를 보유.
- 정합성 검증에서 콘텐츠 누락 **0건**.
- 참고 과목(prompt·spring-ai·rag)도 동일 깊이로 작성 — 일정 미배정이어도 강의안으로 활용 가능.

### 2-2. 왕초보 적합성 (18/20)
- 전문용어를 처음 등장 시 쉬운 말로 풀이(예: "임베딩 = 의미를 담은 숫자 벡터").
- 일상 비유 적극 활용(예: LangChain="자주 쓰는 작업을 레시피 카드처럼 모아둔 부엌").
- 개선점: 일부 고난도 과목(transformer·sllm)은 비유 후에도 수식 직관이 빠를 수 있어, 도입부 1줄 더 완만하게 가능.

### 2-3. 실습 중심성 (19/20)
- 메인 실습 평균 **9.2단계** — 클릭·명령·함수 수준의 "그대로 따라 하기" 구성, 기대 결과까지 명시.
- 과목당 추가 Lab 2~3개(총 **111개**), 코드 블록 **140개**(예제 88 + 실전 52).
- 오전·오후 교시 각각 실습 배치로 손 움직이는 비중 확보.

### 2-4. 코드 주석 품질 (13/15)
- 전체 코드 **60%(1,574/2,630줄)**가 주석 — import·변수·로직·출력 줄에 "무엇을·왜" 한국어 설명.
- 과목 편차: 높은 편(git 80·serving 76·sllm 75·rag 75) / 낮은 편(vue 36·spring-ai 50).
- vue는 절대 주석 줄(124)이 최다지만 SFC 템플릿 구조 줄이 많아 비율이 낮게 보임.

### 2-5. 한국어 가독성 (14/15)
- 심화이론 본문 **109개 100%** 마침표 기준 문장별 줄바꿈 적용, 렌더러 `white-space: pre-line` 반영.
- 이어지는 문장은 문단으로 유지해 의미 단위 보존.

### 2-6. 일관성·구조 (9/10)
- 전 과목 동일한 8교시(09:00~17:50·50분 단위) 시간표와 데이터 양식.
- 강의안↔일정↔진도 페이지가 동일 데이터원(`subjectId-day`)으로 연결돼 정합.

## 4. 강점

1. **완전성** — 42 day-key 빠짐없이 6개 콘텐츠 차원을 채움.
2. **실습 우선** — 따라 하기 단계·Lab·실행 코드 중심, 강의보다 손.
3. **친절함** — 비유·용어풀이로 비전공 왕초보 진입장벽 완화.
4. **가독성** — 한국어 문장별 줄바꿈으로 긴 설명도 술술 읽힘.
5. **일관성·유지보수성** — 데이터 단일원·결정적 재생성 파이프라인.
