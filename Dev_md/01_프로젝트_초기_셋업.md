# 01. 프로젝트 초기 셋업

## 개요
빈 저장소(`CNAME` + `README`만 존재)에서 SKALA 4기 커리큘럼 뷰어를 신규 구축했다.
- 용도: 4기 학습자에게 **과목별 · 일자별 학습내용** 제공
- 스택: React 18 · Vite 5 · Tailwind 3 · React Router 6 · (옵션) Supabase
- 배포: 커스텀 도메인 `skala.dreamitbiz.com`, `main` 푸시 → GitHub Actions 자동배포

## 작업 내용

### 1) 프로젝트 골격
- `package.json` / `vite.config.js`(base `/`) / Tailwind · PostCSS 설정
- `index.html` — Pretendard 폰트, OG 메타태그
- 라우팅: `/`(홈) · `/subjects`(과목별) · `/schedule`(일정) · `/day/:date`(일자별 상세)
- 공통: `Header` · `Footer` · `ScrollToTop`

### 2) 다크 블루 5색 팔레트
`tailwind.config.js` 에 정의 — navy `#0A1F44` · ocean `#14346E` · azure `#2563EB` · sky `#38BDF8` · amber `#FBBF24`

### 3) 커리큘럼 데이터 모델 (`src/data`)
- `course.js` — 과정 메타 + Overview(기간/시간/형태)
- `curriculum.js` — `subjects`(과목) + `sessions`(일자별 수업) + 조회 헬퍼
  - ⚠️ 현재는 **샘플** 데이터. 참고자료 기준으로 교체 예정.

### 4) 참고자료 업로드 폴더
`reference/excel/`, `reference/images/` (+ 안내 README). 대표님이 엑셀·이미지를 올리면 그 내용으로 커리큘럼을 채운다.

### 5) OG 이미지
`scripts/make-og.mjs` 로 `public/og.png`(1200×630, 다크 블루) 생성. sharp 는 임시 설치 후 제거.

### 6) Supabase
`src/lib/supabase.js` — 환경변수(`VITE_SUPABASE_*`)가 있을 때만 클라이언트 생성. 없으면 정적 데이터로 정상 동작.

## 검증
- `npm run build` 성공 (43 modules, ~178KB JS)
- `dist/` 에 `CNAME`, `og.png` 정상 포함
