## 목표

"모든 일정의 강의안·내용을 왕초보도 이해할 만큼 상세하고, 실습에 초점을 맞춰" 전면 재작성.
담당 14과목 + 참고 3과목(prompt·spring-ai·rag) = **18과목 / 42 day-key** 전체.

## 방식 — 멀티에이전트 워크플로 + 결정적 조립

파일 쓰기 충돌과 컨텍스트 폭증을 피하기 위해 2단계로 분리:

1. **생성(워크플로)**: 과목당 에이전트 1개가 과목 스펙(curriculum)을 읽고, 왕초보·실습중심
   강의안을 `/tmp/skala-build/<id>.json`에 기록(과목별 파일 분리 → 충돌 0).
   - 1차 18개 동시 실행 → 12개가 "Connection closed mid-response"로 실패(과부하)
   - 5개씩 배치로 제한해 누락 12개 재실행 → 전원 성공
   - 누적 약 106만 토큰(서브에이전트)
2. **조립(Node 스크립트)**: 18개 JSON을 모아 6개 `.js` 파일을 결정적으로 재생성.
   - 헬퍼(`xxxFor`)·`PERIOD_TIMES`·`examplesExtra` 머지 로직 모두 보존
   - `plan.schedule`은 교시표(periods)와 자동 동기화 생성

## 보강 내용(각 day-key)

| 항목 | 내용 |
|---|---|
| 교시표(periods) | 8교시, 오전·오후 각각 실습 교시 포함 |
| 실습(practice) | 6~10단계의 "그대로 따라 하면 되는" 구체 단계 + 산출물 |
| 핵심개념 | 5~7개, 일상 비유로 쉬운 한 문장 설명 |
| 심화이론 | 2~3개 서술형(왕초보용 설명) |
| 실습예제 | 2~3개 짧고 바로 실행되는 코드(한국어 주석) |
| 실전소스 | 1~2개 엔드투엔드 코드 |
| 상세항목/Lab/과제 | topics 3 · labs 2~3(단계별) · homework 1~2 |

- 코드 언어는 과목별 적합하게(python/vue/bash/java).
- `git`(07/14)은 직전 확정한 **오전3 팀빌딩 / 오후5 Git** 교시 구성을 그대로 보존하고 깊이만 보강.
