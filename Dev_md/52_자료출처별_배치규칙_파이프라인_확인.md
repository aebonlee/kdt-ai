# 52. 자료 출처별 배치 규칙 — 파이프라인 확인 결과 (구조 변경 없음)

**작업일**: 2026-07-21
**지시**: 주강사(전임교수) 자료는 강의안 데이터, 실습교수 자료는 실습교안에 배치. 최진철 울산 5종을 실습교안 쪽으로 이동.

## 1. 결론 먼저 — 이동할 것이 없다. 이미 실습교안에 들어가 있다.

**실습교안은 별도 데이터 소스가 없다.** `scripts/build-textbook.mjs`를 읽고 확인한 사실:

```
scripts/build-textbook.mjs 의 import (4~15행)
  curriculum.js / lectureperiods.js / lecturemodes.js
  exams.js(exams, examsAlt) / quizzes.js
  othercontent.js(otherCourses) / otherdeep.js(otherDeep) / otherexams.js(otherExams)
  othersessions.js
  + 72행: await import(`../src/data/lectures/${id}.js`)   ← split-lectures 생성물
```

즉 **소스는 하나, 렌더러가 둘**이다:

| 렌더러 | 산출물 | 데이터 소스 |
|---|---|---|
| `src/components/EtcCourse.jsx` (6·17행) | 사이트 화면(담당외 과목 상세) | `otherdeep.js` |
| `scripts/build-textbook.mjs` (245행 `otherDeep[id]`) | `public/practice-textbook.html` = **실습교안** | **같은** `otherdeep.js` |

`lectureexamples3`도 동일하다 — `examplesFor`로 합쳐져 사이트 강의안 화면과 실습교안 양쪽에 나간다.

**실증 확인**: Dev_md 51에서 `otherdeep.smartdata`에 넣은 최진철 교수 자료가 실습교안 산출물에 실제로 렌더되는지 grep으로 확인했다.
```
grep -c "SELECT 논리적 처리 순서"  dist/practice-textbook.html  → 1
grep -c "울산 분반 운영 차이"      dist/practice-textbook.html  → 1
```
→ **최진철 울산 5종은 이미 실습교안에 수록되어 있다.** 지시의 목표는 현 배치로 이미 달성된 상태이며, 옮길 대상 자체가 존재하지 않는다.

"실습교안에만" 넣으려면 ① 새 데이터 파일 ② build-textbook의 새 렌더 분기 ③ 스모크의 페이지 수 기준(`실습교안 페이지 31 = 개요+18과목+기타개요+담당외 11과목`) 변경이 모두 필요하다. 지시대로 **임의 구조 변경은 하지 않았고**, 아래 4절에 설계안만 남긴다.

## 2. 주강사/실습교수 판정을 어떻게 했나

세 가지 근거를 교차했다.

1. **파일명 패턴** — 지시가 규정한 실습교수 패턴은 `종합실습가이드_<과목>_(<실습교수명>).pdf` 와 `실습Guide_*.xlsx`
2. **폴더 위치** — `5. 종합실습 교재-가이드(...)/실습교수님 업로드 폴더/` 안에 있으면 실습교수 작성분
3. **슬랙 호칭** — 채널에서 "전임교수/실습교수"로 불린 기록(Dev_md 47·44에 기록됨)

### ⚠️ 최진철 울산 5종은 이 기준으로는 **주강사 자료**다 (지시와 반대)

| 판정 근거 | 결과 |
|---|---|
| 파일명 `[4기] 02_과정안내 교육생배포용(최진철, 울산(D)).pdf` | 실습교수 패턴(`종합실습가이드_..._(이름).pdf`)과 **불일치** |
| 폴더 위치 | `6. 4기 교육생 교재(PDF ver.)/` — **교육생 교재 폴더**, 실습교수 업로드 폴더가 아님 |
| 슬랙 호칭 | Dev_md 47에 **"최진철 전임교수"**로 기록 (울산 운영 공지 주체) |
| 지시문 자체 | 주강사 예시로 "백정열, 박병선w, 권기창, **최진철** 등"을 들면서, 뒤에서 같은 최진철의 울산 5종을 실습교수 자료로 지목 — **내부 상충** |

세 근거가 모두 주강사를 가리키고 지시문 안에서도 상충하므로, **현 배치(강의안 데이터)가 규칙에 맞다**고 판단했다. 어느 쪽으로 판정하든 1절 때문에 **결과 배치는 동일**하다.

## 4. 구조 제안 (필요하다고 판단되면 그때 실행)

출처별 **분리 노출**이 실제 요구라면, 데이터 이동이 아니라 **메타데이터 + 렌더 분기**가 최소 변경이다.

- **A안 (권장·최소)** — 카드에 선택 필드 `origin: 'practice'` 추가(없으면 기존과 동일 동작). `build-textbook.mjs`가 해당 카드를 과목 말미 "실습교수 보충자료" 소절로 묶어 렌더. 화면에서도 배지 하나만 붙이면 되고, **데이터 이동 0건·스모크 기준 불변**.
- **B안** — `src/data/practiceonly.js` 신설 + build-textbook 전용 섹션. 화면에서는 안 보이게 됨. 스모크 페이지 수 기준 수정 필요.
- **C안** — 현행 유지(제목 괄호 표기로만 구분). 지금 상태.

A안은 반나절, B안은 스모크·페이지 구조까지 손대야 해 하루 이상으로 본다. **대표 판단 대기.**
