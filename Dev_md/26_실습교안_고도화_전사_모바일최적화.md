# 26. 실습교안 뷰어 고도화 · 전사 모바일 최적화 · 로그인 재디자인 · 선수/후속 안내

## 1) 실습교안 뷰어 고도화 (scripts/build-textbook.mjs)
Dev_md/24의 웹 뷰어를 학습 도구 수준으로 발전:
- **페이지 단위 구성**: 좌측 메뉴(과목) 클릭 → 해당 과목만 표시(개요+18과목=19페이지). 무JS 환경은 전체 스크롤 폴백. 페이지 섹션의 id 제거로 해시 진입 시 네이티브 앵커 점프(상단 빈공간) 방지 — 내비게이션은 JS `show()` 전담
- **인쇄**: 상단 툴바 "이 과목 인쇄" 버튼 → `@media print`로 현재 과목만 출력
- **코드 녹색 주석**: 사이트 CodeBlock 감지 로직(hlCode) 이식 — 언어별 라인 주석 토큰(#, //, --, `<!-- -->`)을 `.cmt`(#4ade80)로 표기
- **소스 복사 버튼**: 블록별 복사(clipboard + execCommand 폴백), 인쇄본 숨김
- **모바일(≤820px)**: 상시 목차 제거 → **☰ 햄버거 + 오프캔버스 드로어**(오버레이·ESC·선택 시 닫힘·배경 스크롤 잠금). 코드는 **기본 줄바꿈**(제스처 불필요) + 블록별 "가로↔줄바꿈" 토글(ASCII 다이어그램 대응)
- 명칭: "강의안 교재" → **"실습 교재"**, 표기 "담당 18과목". PDF 파일명 `SKALA_4기_실습교재_이애본.pdf`
- 버그 수정: 칩(배지) 세로 늘어짐(flex align-items:center + inline-flex), 모바일 그리드 min-content 오버플로(minmax(0,1fr))

## 2) 전사(React 사이트) 모바일 최적화 — 측정 기반
각 라우트에 scrollWidth 측정 프로브를 심어 **실제 가로 오버플로만** 확정 후 수정 (vw=500 기준):

| 페이지 | 전 | 원인 | 후 |
|---|---|---|---|
| 수업일정표 | 836 | 캘린더 `repeat(7,1fr)` min-content 바닥 | 500 OK |
| 학습강의안 | 744 | `.layout-side` 모바일 1fr + 칩 | 500 OK |
| 팀 프로젝트 | 687 | 동일 | 500 OK |
| 나머지 6개 라우트 | 500 | 정상 | — |

수정: `.cal` → `repeat(7, minmax(0,1fr))` + `.cal-cell{min-width:0}` / `.layout-side` 모바일 `minmax(0,1fr)` + 자식 `min-width:0` / 전역 안전장치 `html{overflow-x:clip}`(sticky 유지하며 잔여 가로스크롤 차단). 관리자 페이지·풍선메뉴(right:0, 232px)는 기존 구조+clip 가드로 안전 확인.
