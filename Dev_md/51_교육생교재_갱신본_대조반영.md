## 1. 우선순위 1 — 오늘 반영분 대조 결과

### 1-1. Prompt 교재(박병선w) ↔ 오늘 커밋 `c5627ea`의 6카드

**결론: 충돌 없음. 정정 불필요. 추가 반영분도 없음.**

- 교재는 **97쪽 전부 이미지 슬라이드**로, 텍스트 레이어가 4.8KB뿐이고 대부분 `SKALA` 워터마크 조각·불릿 기호다.
- 추출 가능한 실질 내용은 3개뿐이었고 **셋 다 기존 카드에 이미 반영되어 있었다**:

| 교재에서 추출된 내용 | 기존 카드 |
|---|---|
| 사실 고정 모드 / 아이디어 탐색 모드 프롬프트 전문 | `prompt-1` "사실 고정 모드 vs 아이디어 탐색 모드" |
| Context 부품 — 설계·메모리·히스토리·도구·외부지식·포맷 | `prompt-1` "Context Engineering 부품 4개 점검" |
| `ai-harness/` 폴더 트리(AGENTS.md·docs·templates·tools·outputs·logs) | `prompt-1` "AI 하네스(작업장) 폴더 만들기" — **트리 구조가 그대로 일치** |

→ 기존 prompt 카드들이 **이 교재와 같은 박병선 계열에서 나온 것**임이 확인됐다. 오늘 추가한 6카드는 **임성열 종합실습 정본** 계열이라 트랙 자체가 달라 겹치지 않는다(교재=이론, 정본=종합실습). 정합성 문제 없음.

### 1-2. ⚠️ 팀 인원 — 이 교재에서는 확인 불가

- 텍스트 레이어에 `팀`·`조별`·`N인`·`개인 실습` **어떤 표기도 없다**(grep 0건).
- 애초에 팀 인원은 **종합실습 가이드에 적히는 항목**이고 이 파일은 **이론 강의 교재**라 없는 것이 자연스럽다.
- 즉 Dev_md 50에서 보고한 불일치(**임성열 정본 3인 / 하만석 2인 1조 / 박영웅 개인**)는 **이 교재로 해소되지 않는다.** 여전히 대표 확인이 필요하다.
- 이미지 슬라이드라 육안 판독은 가능하나, 판독 결과를 근거로 쓰는 것은 오독 위험이 있어 하지 않았다. 필요하면 대표가 해당 PDF를 직접 넘겨 확인하는 편이 빠르다.

### 1-3. Python 교재(백정열) 7/15 → 7/16 변경점

`diff` 결과 **실질 변경 3건뿐**(diff 53줄, 전부 아래에 해당). 정오표 성격이다.

| 변경 | 내용 | 상태 |
|---|---|---|
| ① 제출 파일명 규칙 | `이름_실습명.zip` → **`캠퍼스명_반_이름_실습명.zip`** (예: 서울_1반_홍길동_day1종합실습.zip), Practice는 `캠퍼스명_반_이름.py` | **신규 반영** ✅ |

②③은 슬랙 공지로 먼저 반영했던 것이 교재 공식본에 그대로 반영된 것으로, **선반영이 정확했음이 사후 검증**됐다.

**⚠️ 운영 함정 1건 (대표 인지 필요)**
